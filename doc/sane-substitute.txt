FEATURES                  *sane-substitute-nvim-sane-substitute-ðŸ˜Œ-features*


- Use `:SaneSubstitute` or the short form `:S` to do perform search-and-replace
    operations, in lua patterns or javascript regex.
- Supports ranges, if no range is given, works on the entire buffer (`%s`)
- The `g` flag is supported and works like with `:substitute`. Without the `g`
    flag, only the first match per line is replaced.
- Incremental preview of your substitution.
- Support for more flavors is easy to add. |sane-substitute-pull-requests-adding-more-regex-flavors-are-welcome.|

An example of `:substitute` and `:SaneSubstitute` that does the same thing,
removing digits appended to any word in the entire buffer.

>lua
    -- intended replacement operation
    deviceModel2020 -> deviceModel
    
    -- vim's :substitute
    :%s /\(\w\+\)\d*/\1/g
    
    -- nvim-sane-substitute (using lua flavor)
    :S (%w+)%d*/%1/g
<

CONFIGURATION        *sane-substitute-nvim-sane-substitute-ðŸ˜Œ-configuration*

>lua
    -- default values
    opts = {
        regexFlavor = "lua", -- currently only supports "lua" and "javascript"

<

Note that any regex flavor other than `"lua"` requires the respective language
support to be installed on your machine. `"javascript"`, for instance, requires
`node`.


SUPPORT FOR MORE FLAVORS*sane-substitute-nvim-sane-substitute-ðŸ˜Œ-support-for-more-flavors*

The plugin has been specifically build in with easy extensibility in mind. It
should take no more than ~10 LoC to add support for more regex flavors. Have a
look at how javascript regex is supported <./lua/regex/flavors/javascript.lua>.


LIMITATIONS            *sane-substitute-nvim-sane-substitute-ðŸ˜Œ-limitations*


- Only the `g` flag is supported.
- Does not support `inccommand=split`. Please use `inccommand=unsplit` instead.
